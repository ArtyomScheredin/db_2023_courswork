Что такое внешний ключ?

Внешний ключ - это атрибут таблицы, который ссылается на значения из другой таблицы, называемой родительской.
Он используется для создания связи между таблицами и поддержания целостности данных.

 Какие существуют способы поддержания ссылочной целостности?
Существуют несколько способов поддержания ссылочной целостности: ограничение ссылочной целостности, триггеры и хранимые процедуры.

Что такое уникальный ключ?

Уникальный ключ - это ограничение целостности, которое гарантирует, что значения в столбце
 или группе столбцов уникальны в пределах таблицы.

 Что такое SERIAL?

SERIAL - это псевдоним для типа данных INTEGER, который автоматически
 генерирует уникальные числовые значения при вставке новой строки.

 Рассказать о значениях по умолчанию и неопределенных значениях.

По умолчанию, если значение не задано при вставке новой строки, то используется значение по умолчанию для соответствующего типа данных.

Как можно хранить даты и время?

Для значений даты и времени можно использовать тип данных DATE, TIMESTAMP или TIMESTAMPTZ.

 Рассказать о числовых типах данных.

Числовые типы данных включают INTEGER, BIGINT, SMALLINT, NUMERIC, DECIMAL, REAL и DOUBLE PRECISION.

 Каким образом можно вставить несколько строк с помощью одного
оператора INSERT?

Для вставки нескольких строк одним оператором INSERT можно использовать конструкцию VALUES.
 Например: INSERT INTO mytable (col1, col2) VALUES (1, 'a'), (2, 'b'), (3, 'c').


 Как ведет себя оператор INSERT, если в списке столбцов перечислены не все
столбцы?

Если в списке столбцов не перечислены все столбцы, то оператор INSERT заполнит недостающие столбцы значениями NULL
или значениями по умолчанию, если они определены.

 Добавить какие-либо ограничения целостности.

Ограничения целостности могут включать ограничения уникальности, ограничения проверки,
 ограничения ссылочной целостности и другие. Они могут быть добавлены при создании таблицы
 или внесены позже с помощью оператора ALTER TABLE.
