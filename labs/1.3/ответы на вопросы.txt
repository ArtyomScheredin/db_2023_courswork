Рассказать про операцию соединения (JOIN) и различные её разновидности.

JOIN - это операция объединения двух или более таблиц на основе определенного условия соединения.
Существуют различные типы JOIN, включая INNER JOIN, LEFT JOIN, RIGHT JOIN и FULL OUTER JOIN,
каждый из которых определяет, какие строки из таблицы будут включены в результат соединения.

Рассказать про агрегатные функции, предложения GROUP BY и HAVING.

Агрегатные функции - это функции, которые применяются к группе строк и возвращают
единственное значение, такое как COUNT, SUM, AVG, MIN и MAX.
GROUP BY используется для группировки строк по одному или нескольким столбцам,
а HAVING определяет условие, которое фильтрует группы строк.

Как выбрать только уникальные значения какого-либо столбца?

Используйте ключевое слово DISTINCT, за которым следует имя столбца,
для выбора только уникальных значений. Например: SELECT DISTINCT column_name FROM table_name;

Как осуществить сортировку по возрастанию/убыванию по значению какого-либо столбца?

Используйте ключевое слово ORDER BY, за которым следует имя столбца и ключевое слово ASC или DESC,
чтобы отсортировать столбец по возрастанию или убыванию соответственно.
Например: SELECT column_name FROM table_name ORDER BY column_name ASC;

Как агрегатные функции ведут себя по отношению к неопределённым значениям?

Агрегатные функции игнорируют неопределенные значения при вычислении результата.

Рассказать о теоретико-множественных операциях в SQL.
Теоретико-множественные операции в SQL включают UNION,
UNION ALL, INTERSECT и EXCEPT, которые выполняют операции объединения,
пересечения и разности между наборами строк из одной или нескольких таблиц.

Чем отличаются UNION и UNION ALL?

UNION объединяет наборы строк из двух или более таблиц
и удаляет дубликаты, тогда как UNION ALL объединяет наборы строк,
 сохраняя все строки, включая дубликаты.

Чем отличаются COUNT(*) и COUNT(field)?
COUNT(*) возвращает количество строк в таблице,
тогда как COUNT(field) возвращает количество строк,
в которых поле field имеет значение, отличное от NULL.

Как подсчитать количество уникальных значений столбца?

Используйте функцию COUNT()

Округлить результирующее значение до 3 знаков после точки.

Для округления числа до 3 знаков после запятой в PostgreSQL
используется функция round, которой передаются два аргумента: число и количество знаков после
запятой до которых нужно округлить. Например:
SELECT round(1.23456789, 3);


Округлить вещественное число до целого без нулей после точки.

Для округления вещественного числа до целого без нулей после запятой
в PostgreSQL можно использовать функцию cast, приводящую число к типу integer. Например:
SELECT cast(1.23456789 as integer);


Переписать запрос, не используя функцию MAX (MIN).
Для нахождения максимального или минимального значения
в столбце таблицы можно использовать подзапрос с оператором
ALL и оператором сравнения. Например, для поиска
максимального значения в столбце "price" таблицы "products" можно написать следующий запрос:
SELECT price
FROM products
WHERE price >= ALL (SELECT price FROM products);

Изменить формат вывода данных (например, формат даты и времени).

Для изменения формата вывода данных в PostgreSQL
используется функция to_char, которая позволяет преобразовать
значение определенного типа данных в строку с заданным форматом.
Например, для вывода даты в формате "день.месяц.год" можно использовать следующий запрос:
SELECT to_char(date_column, 'DD.MM.YYYY') FROM table_name;
